// how it was built

What I solved

The platform was already live, so every fix had to hold up in production, not just on localhost.

  • Broken Fanzone tab navigation — caused by missing type="button" attributes on form buttons, fixed across the module.
  • Fatal PHP redeclaration error — tracked a function being declared twice in includes/db.php, breaking every page that loaded it.
  • Rebuilt the Fanzone bill flow — moved it to AJAX, backed by a new fanzone/get_bill_items.php endpoint, so bills update without a page reload.
  • Live-server path bugs — the /dewol/ prefix on the live host was silently breaking file_exists(), unlink(), and fetch() calls that worked fine locally.
  • Hardcoded JS fetch paths — replaced with a window.SITE_URL value injected from includes/header.php, so the same JS works on localhost and live.
  • Missing AJAX handlers — created four missing handler files across the water, homes, and Fanzone modules.
  • Rebuilt the email system — includes/notify.php now triggers manager alerts after in-app notifications, every AJAX handler sends customer status-change emails, and a new sendCustomerAcknowledgment() function was added to includes/mailer.php.
